# Operations Checklist Scan

Update an operations checklist from clear completion evidence.

Rules:

1. Read the checklist and recent evidence source.
2. Treat ambiguous notes as unresolved.
3. Preserve safety/escalation notes and problem reports.
4. Commit only checklist changes in the owning surface.
5. Report what was marked complete, what remains, and what needs clarification.

Closeout ratchet: read `hub/MEMORY/LESSONS.md` before acting. Before ending,
log the run, leave one improvement (lesson, correction, or pruning), refresh
memory health, and close the run record with `hub/scripts/run_close.py`.
