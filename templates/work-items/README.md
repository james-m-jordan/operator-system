# Work Items

This folder contains canonical execution packages for `{{ORG_NAME}}`.

Each work item should have enough context for an agent or collaborator to act
without guessing:

- owner or requester
- requested outcome
- source files or authoritative inputs
- metadata that maps source files, rows, sheets, attachments, or messages to
  meaning
- constraints, deadlines, approval boundaries, and next actions

Suggested structure:

```text
work-items/<id>/
  README.md
  source/
  metadata/
  analysis/
  outputs/
  decisions.md
  next-actions.md
```
