# Release Checklist

Use this before packaging or handing the starter kit to another organization.

## Required Checks

```bash
python3 -m unittest tests/test_scaffold_workspace.py
python3 -m py_compile scripts/*.py templates/hub/scripts/*.py
python3 scripts/audit_release.py --write COMPLETION-MATRIX.md
```

From the repository root used by the live workspace, the equivalent commands are:

```bash
python3 -m unittest hub/distribution/operator-system/tests/test_scaffold_workspace.py
python3 -m py_compile hub/distribution/operator-system/scripts/*.py hub/distribution/operator-system/templates/hub/scripts/*.py
python3 hub/distribution/operator-system/scripts/audit_release.py --write COMPLETION-MATRIX.md
```

## Manual Smoke Test

```bash
rm -rf /tmp/operator-system-demo
python3 scripts/scaffold_workspace.py --config config/org.example.json --out /tmp/operator-system-demo --force
cd /tmp/operator-system-demo
python3 hub/scripts/preflight_capabilities.py --root .
python3 hub/scripts/sync_workspace.py --root .
python3 hub/scripts/memory_index_refresh.py --root . --write --validate
python3 hub/scripts/state_digest.py --root .
python3 hub/scripts/install_automations.py --root . --out .operator-automations
python3 hub/scripts/export_runtime_adapters.py --root .
python3 hub/scripts/run_automation.py --root . --automation-id morning-control-panel
python3 hub/scripts/publish_status.py --root . --publisher status --message "Ready for review"
python3 hub/scripts/deliver_outbox.py --root . --publisher status --latest
python3 hub/scripts/backup_transfer.py --root . --write-report
printf 'example upload\n' > /tmp/operator-upload.txt
python3 hub/scripts/chat_file_fetch.py --root . --provider direct-url --url file:///tmp/operator-upload.txt
```

Add a sample work item and run `package_gate.py` before claiming package-gate
coverage for a release.

## Build Archive

```bash
python3 scripts/package_release.py --version YYYY-MM-DD --out dist
```

The packager writes:

- `operator-system-starter-<version>.tar.gz`
- `operator-system-starter-<version>.manifest.json`

Check the manifest before handoff. It records file count, file hashes, and the
archive hash.

## Completion Matrix

`COMPLETION-MATRIX.md` is generated by `scripts/audit_release.py`. It should
show `Audit result: PASS` before packaging or handoff. The audit verifies
required release files, generated helper scripts, automation manifest coverage,
config surfaces, generated workspace neutrality, and archive contents.

## Review Gates

- No generated file contains live organization names, private people, or
  hard-coded source paths.
- `local-private/`, credentials, and private packets remain ignored.
- Runtime adapter files are documented as drafts that require operator review.
- Live delivery and backup transfer helpers default to dry-run unless
  `--execute` is passed.
- Chat file fetches land in ignored `local-private/` before any canonical work
  item intake.
- The release notes name any missing provider-native connector adapters.
- The package includes `README.md`, `INSTALL.md`, `MIGRATION.md`,
  `BLUEPRINT.md`, `RELEASE-CHECKLIST.md`, `COMPLETION-MATRIX.md`, `config/`,
  `templates/`, `scripts/`, and `tests/`.
